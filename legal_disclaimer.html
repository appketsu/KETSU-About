KETSU is an interpreter of modules.

KETSU is a tool that allows to adapt websites or utilities into KETSU via modules (Scripts), this modules contain the instructions for the app to work. KETSU doesn´t provide any modules nor promotes them.

KETSU by itself can´t do anything unless you load a module on it, modules make the app function.

KETSU can be compared to a video player or an pdf viewer, this apps allows you to feed them pdf or videos so they can be used. KETSU is no different than this apps, but the difference is that it uses its own format called modules.

KETSU modules don´t run on any external server, everything is executed on the users phone through different apis provided by apple to interact with the WKWebview such evaluateJavascript()

The idea of KETSU is to be an app that adapts websites into app format, via modules. Offering all the extra features that KETSU has which make the interaction for the users more appealing and complete.

The developer of ketsu is not responsible for any of the modules, KETSU doesn't distribute modules. ketsu by itself can't do anything besides load and intepret Modules. The code of the modules is in JSON format.

Modules can be created and imported by anyone. The user is responsible of the modules that he decide to load on KETSU.

KETSU provides documentation so the users can create their own modules for KETSU, the documentation showcase the modules system and how to create a module.

Modules are just pieces of instructions that KETSU follow to display data.

KETSU doesn't host or provide any type of content to the user, the Modules that the user decide to load will display the content. The user decides the modules that he wants to load on KETSU, the developer of KETSU is not responsible for the modules that the user decide to load on KETSU.
